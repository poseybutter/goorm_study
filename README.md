## 스터디 가이드라인

1 . 레포지토리 Fork
- 스터디에 참여하기 위해 `goorm_study` 레포지토리를 자신의 계정으로 `fork` 합니다.

2 . 개인 브랜치에서 작업
- 모든 자료 업로드 및 수정은 개인 브랜치에서 진행합니다.
- 브랜치 명은 `feature/영어이름` 형식으로 작성합니다.
- 예: `feature/kyg`

3 . Pull Request 작성
- 개인 브랜치에서 작업이 완료되면 원본 레포지토리로 `PR(Pull Request)`을 제출합니다.
- PR 제목과 내용은 작업한 회차와 자료 내용을 명확히 작성합니다.

4 . 관리자 Merge

- 제출된 PR은 관리자가 검토 후 `main` 브랜치로 병합합니다.

- PR 승인 전에는 `main` 브랜치에 직접 반영하지 않습니다.

<br>

### 커밋 메세지
```
[0회차] 제목/간단설명
```
- 예시: `[1회차] 배열 기본 개념 정리`

<br>

### PR 작성
```
[0회차] 이름_문서 링크
```
- 예시: `[1회차] 김영규_문서 링크 업로드`

<br>

### Issue 등록
```
[날짜] 1회차_스터디
```
- 예시 : [260110] 1회차_스터디
